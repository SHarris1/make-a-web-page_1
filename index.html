Lesson 1: The Basics of the Web and HTML

How the Web Works

The web is a collection of html pages ie links. A web page is a text doc written in a language called HTML. Web browsers read these documents, and then interpret and display them. Here is a visual {me -> computer + browser -> the internet -> /http/ ->Servers (then back to me)}. This Video does a good job of explaining that flow of information. HTML = hyper text markup language. HTML documents contain text content = what I see and markup = how it looks.

Tags and Elements
  1. HTML Tags: are always contained within angled brackets.
  2. Most tags have an opening tag (<p> for example) and a closing tag, (</p>)
  3. Some tags (called "void" tags) do not require a closing tag (like the <br> tag)r
  4. Element: An HTML element refers to everything within a set of opening and closing tags

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them. Inline includes: b, span, br, img, strong, a. Block includes: div, p, form

Chrome Developer Tools

Use chrome to see html/structure and css/style on any web page. Go to settings, tools, developer tools to get split screen bottom tools. Tree like structure, from top to bottom, looking at 'play arrows' (then expand them to see more).

Important takeaways: Dom, Tree like structures, HTML and CSS
  • HTML and CSS are both "languages"
  • HTML controls the "structure" of a web page
  • CSS controls the "style" of a page (how it looks).
  • When programmers talk about the "DOM" (Document Object Model) they are talking about the tree-like structure of a page.
  • To learn more about what a "tree-like structure" means, read the first two paragraphs of the wikipedia article on Tree Structures.
  • Websites and html are broken into boxes (images, text, headers, footers, etc). Define boxes with 'div' tags.

Lesson 3: Adding CSS Style to HTML Structure

CSS & Avoiding Repetition

CSS allows programmers to avoid repetition. They do this by using refrential data such as CSS to apply style to large and numerous blocks of elements/tags. The alternative would be to apply style at each element...eventually it gets ridiculous and a major pain when updating things...having to change it in a million places instead of a handlful. CSS stands for "Cascading Style Sheets." The "C" is by far the most important part of this phrase.That idea is known as inheritance and understanding it requires abstract thinking.

Treat yourself...don't REPEAT yourself!!

In general, descendant elements inherit text-related properties, but box-related properties are not inherited.

  • Properties that CAN BE inherited: are color, font, letter-spacing, line-height, list-style, text-align, text-indent, text-transform, visibility, white-space and word-spacing
  • Properties that CANNOT BE inherited are background, border, display, float and clear, height and width, margin, min- and max-height and -width, outline, overflow, padding, position, text-decoration, vertical-align and z-index.
  • Inheritance prevents certain properties from being declared over and over again in a style sheet, allowing the software developers to write less CSS. It enhances faster-loading of web pages by users and enables the clients to save money on bandwidth and development costs.

Box Sizing & Positioning

There are four main points that Jessica addressed about box sizing:

  1. HTML elements are boxes and each box has 4 components.
  2. Because there are so many components to each box, it can often be hard to get the size of a box just right.
  3. There are two techniques you can use to help deal with sizing issues.
    • 3a. Set sizes in terms of percent rather than pixels.
    • 3b. Set the box-sizing attribute to border-box for every element.
  4. 4. Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently.

Box Positioning

  1. Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.
  2. Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other. Not exactly sure how this works...need to revisit or find sample.
